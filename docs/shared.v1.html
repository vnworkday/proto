message Error shared/v1/error.proto

Field Type Description
1 code int32

Required. Error code that can be used to identify the error type.
2 message string

Optional. Error message that can be used to display the error to the user.
3 retryable bool

Required. Whether the error is retryable or not. Default is false.
4 reason string

Optional. Reason for the error. This can be used to provide more context to the error. Should NOT be displayed to the user.

Field Type Description

message RequestFilter shared/v1/request.proto

Field Type Description
1 field string

Required. The field to filter by.
2 value string

Conditionally required. The value to filter by.
If the operator is 'null' or 'not_null', the value will be ignored and can be omitted.
If the operator is 'in' or 'not_in', the value should be a list of values separated by vertical bars '|'.
3 operator shared.v1.Operator

Required. The operator to use for the filter.
4 is_case_sensitive bool

Optional. Whether the filter is case sensitive. The default is false.
5 value_type shared.v1.ValueType

Optional. The type of the value. The default is string.

message RequestPagination shared/v1/request.proto

The request to fetch a list of items.

Field Type Description
1 token string

Optional. The token that should be used to fetch the next page. If not provided, the first page will be fetched. In form of encoded base64 string.
2 limit int32

Optional. The limit of the number of items to return. The default is 10.
3 filters repeated shared.v1.RequestFilter

Optional. The filters to apply to the items. The default is no filters.
4 sorts repeated shared.v1.RequestSort

Optional. The sorts to apply to the items. The default is to sort by the last_modified_date field in ascending order or no sort if the field is not present.

message RequestSort shared/v1/request.proto

Field Type Description
1 field string

Required. The field to sort by. The default is the updated_at field.
2 order string

Optional The order of the sort, either 'asc' or 'desc'. The default is 'asc'.

message ResponsePagination shared/v1/response.proto

Field Type Description
1 next_token string

Optional. The token that can be used to fetch the next page. If the token is empty, it means that there are no more pages.
2 previous_token string

Optional. The token that can be used to fetch the previous page. If the token is empty, it means that there are no previous pages.
3 total int32

Required. The total number of items.
4 total_pages int32

Required. The total number of pages.
