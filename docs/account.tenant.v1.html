This package provides a service for managing tenants in the system.
Tenants are the top-level entities in the system, used to separate data and configuration among organizations.

Services

service TenantService account/tenant/v1/tenant_service.proto

Tenant deletion is not supported. Instead, tenants can be disabled by setting their status to INACTIVE.

rpc CreateTenant
Request: account.tenant.v1.CreateTenantRequest
Response: account.tenant.v1.CreateTenantResponse

Create a new tenant with default status set to ACTIVE and default production type set to ENTERPRISE.

rpc GetTenant
Request: account.tenant.v1.GetTenantRequest
Response: account.tenant.v1.GetTenantResponse

Get a tenant by ID.

rpc ListTenants
Request: account.tenant.v1.ListTenantsRequest
Response: account.tenant.v1.ListTenantsResponse

List tenants. Supports pagination and filtering.

rpc UpdateTenant
Request: account.tenant.v1.UpdateTenantRequest
Response: account.tenant.v1.UpdateTenantResponse

Update a tenant. Compared to CreateTenant, the domain, timezone of the tenant cannot be changed.

message CreateTenantRequest account/tenant/v1/tenant_service.proto

Optional. Whether self-registration is enabled for the tenant. Defaults to false if not set.

message ListTenantsRequest account/tenant/v1/tenant_service.proto

For listing tenants.

Field Type Description
1 info shared.v1.RequestInfo
2 pagination shared.v1.RequestPagination
3 filters repeated shared.v1.RequestFilter

Optional. Filters to apply to the list of tenants.
If multiple filters are specified, they are combined with AND.
Supported filters:
- name: The name of the tenant.
- domain: The domain of the tenant.
- timezone: The timezone of the tenant.
- subscription_type: The subscription type of the tenant.
- self_registration_enabled: Whether self-registration is enabled for the tenant.
- status: The status of the tenant.
4 sorts repeated shared.v1.RequestSort

Optional. Sorts to apply to the list of tenants.
If multiple sorts are specified, they are applied in the order given.
Supported sorts:
- name
- domain
- timezone
- subscription_type
- self_registration_enabled
- status
- created_at
- updated_at

message Tenant account/tenant/v1/tenant.proto

A group or an entity that has its own isolated environment and resources within a shared system.

Field Type Description
1 id string

Required. In form of nanoid.
2 name string

Required. Name of the tenant. Unique within the system.
3 status account.tenant.v1.TenantStatus

Required. Default is PROVISIONING status.
4 domain string

Required. Domain of the tenant. Unique within the system. Example: "example.vnworkday.com".
5 timezone string

Required. Timezone of the tenant. Following Olson format. Default is UTC. Example: "America/Los_Angeles", "Asia/Ho_Chi_Minh".
6 production_type account.tenant.v1.TenantProductionType

Required. Default is ENTERPRISE.
7 subscription_type account.tenant.v1.TenantSubscriptionType

Required. Default is BASIC.
8 self_registration_enabled bool

Required. Describe whether the user can self-register to the tenant or must be invited by the tenant admin. Default is false.
10 created_at google.protobuf.Timestamp

Required. Time when the tenant was created.
11 updated_at google.protobuf.Timestamp

Required. Time when the tenant was last updated.

Enums

enum TenantProductionType account/tenant/v1/tenant.proto

Name Number Description
TENANT_PRODUCTION_TYPE_UNSPECIFIED 0
TENANT_PRODUCTION_TYPE_ENTERPRISE 1

Default. Enterprise tenant is a tenant that is created by an enterprise customer.
TENANT_PRODUCTION_TYPE_ROOT 2

Root tenant is a tenant that is created by the system itself. It is used for system-wide configurations. There is only one root tenant in the system.
TENANT_PRODUCTION_TYPE_INTERNAL 3

Internal tenant is a tenant that is created by the system itself. It is used for internal testing purposes.

enum TenantStatus account/tenant/v1/tenant.proto

Name Number Description
TENANT_STATUS_UNSPECIFIED 0
TENANT_STATUS_PROVISIONING 1

Default. After the tenant is created, it is placed in the provisioning state and not ready for use yet.
TENANT_STATUS_ACTIVE 2

Active tenant is a tenant that is ready for use and can be accessed by users.
TENANT_STATUS_INACTIVE 3

Inactive tenant is a tenant that is not ready for use and cannot be accessed by users. It is disabled by the tenant admin only.

enum TenantSubscriptionType account/tenant/v1/tenant.proto

Name Number Description
TENANT_SUBSCRIPTION_TYPE_UNSPECIFIED 0
TENANT_SUBSCRIPTION_TYPE_BASIC 1

Default. Basic subscription is a subscription that provides basic features with most affordable pricing.
TENANT_SUBSCRIPTION_TYPE_STANDARD 2

Standard subscription is a subscription that provides standard features with advanced pricing.
TENANT_SUBSCRIPTION_TYPE_PREMIUM 3

Premium subscription is a subscription that provides premium features with the highest pricing.
